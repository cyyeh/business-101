# 內容驗證日誌（Verification Log）

> 本檔案追蹤本站內容的事實查證進度。每個經過查證的主題都會在此留下紀錄，包含查證日期、查證者、新增的參考資料、未解決的疑點與發現的事實錯誤。

## 流程說明

每次驗證一個主題時，請在「驗證紀錄」區塊新增一筆。流程為：

1. **抽取主張（Claim Extraction）** — 通讀目標檔案，列出所有可查證的事實主張（具體數字、日期、機構名稱、命名框架、歸屬性陳述）。
2. **查證（Search）** — 使用 WebSearch / WebFetch 找原始來源，優先 Tier 1（官方機構、監管單位、原始公司文件）→ Tier 2（主流財經媒體、CFI / Investopedia / HBR）→ Tier 3（Wikipedia，僅作起點且須與更高等級來源並列）。
3. **引用（Cite）** — 在主張後方加上 Unicode 上標連結（如 `[¹](#ref-fs-1)`），並在文末新增 `## 參考資料` 區塊，使用 `<a id="ref-xxx-N"></a>N. [來源](URL) — 說明` 格式。
4. **記錄（Log）** — 在下方表格新增一筆紀錄。
