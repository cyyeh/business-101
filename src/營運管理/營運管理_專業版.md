# 營運管理：專業版

## 一、營運管理的定義與範疇

營運管理（Operations Management）是一門致力於將組織的資源轉換為產品與服務的學科。根據APICS（現為ASCM）的定義，營運管理涵蓋所有計畫、排程與控制物料、資訊與資本流動的活動，以滿足客戶需求。

**核心職能領域包括：**
- **生產運營**（Manufacturing Operations）：製造業的製程管理
- **服務運營**（Service Operations）：醫療、零售、金融等服務業
- **供應鏈管理**（Supply Chain Management）：採購至交付的全流程
- **品質管理**（Quality Management）：持續改善與缺陷預防
- **存貨控制**（Inventory Control）：庫存優化與成本降低

營運管理的成功直接影響企業的成本結構、客戶滿意度與競爭優勢。在當代全球經濟中，有效的營運策略已成為企業策略性競爭力的關鍵。

## 二、流程設計與分析（Process Design & Analysis）

### 2.1 流程基本架構

流程（Process）是一系列有序的活動，將一個或多個投入轉換為預期的產出。優秀的流程設計需要平衡三個維度：

| 維度 | 說明 | 範例 |
|------|------|------|
| **成本效率** | 最小化資源消耗 | 自動化製造、標準化服務 |
| **品質卓越** | 提高產出品質與一致性 | 六標準差、持續改善 |
| **時間敏捷** | 縮短運氣時間，快速反應 | 精實生產、敏捷製造 |

### 2.2 流程映射與分析（Process Mapping）

流程映射（Process Mapping）是通過視覺化方式記錄流程的每一步。常用工具包括：

**價值流圖分析（Value Stream Mapping, VSM）**：標識流程中的增值活動（VA）與非增值活動（NVA）。在製造業中，通常只有10-30%的時間用於增值活動，其餘為等待、搬運等浪費。

**流程瓶頸分析**：識別限制流程效率的環節。根據Goldratt的制約理論（Theory of Constraints），每個系統都有一個瓶頸，管理著整體產出。

### 2.3 流程重新設計（Business Process Reengineering, BPR）

Hammer與Champy於1990年代提出的BPR強調根本性、劇烈性的流程改造。成功案例包括：

- **IBM信用部門**：從7天縮短至4小時的信用審批
- **Kodak影像處理**：通過自動化減少90%的處理人力

## 三、產能規劃（Capacity Planning）

### 3.1 產能的定義與測量

產能（Capacity）是指單位時間內的最大產出量。根據利用率（Utilization Rate）的不同，產能有三種衡量方式：

1. **理論產能**（Design Capacity）：設備在最優條件下的產出量
2. **有效產能**（Effective Capacity）：考慮計畫停機、保養後的產出量（通常為理論產能的80-90%）
3. **實際產能**（Actual Output）：考慮所有現實因素後的實際產出

**產能利用率公式：**
```
利用率(%) = 實際產能 / 有效產能 × 100%
效率(%) = 有效產能 / 理論產能 × 100%
```

### 3.2 產能規劃策略

**追蹤策略（Chase Strategy）**：調整產能以配合需求變化
- 優點：庫存低、靈活性高
- 缺點：頻繁調整成本高、員工波動大
- 適用：勞動密集、需求易預測的行業

**平衡策略（Level Strategy）**：保持穩定的產能
- 優點：員工穩定、成本可控
- 缺點：可能產生庫存或無法滿足需求
- 適用：產能難以快速調整的製造業

**混合策略（Hybrid Strategy）**：結合上述兩種方法

### 3.3 長期產能決策

長期產能決策涉及重大資本投資，需要考慮：
- **需求預測**：用於估計未來產能需求
- **規模經濟**：單位成本隨產能增加而遞減
- **技術選擇**：不同技術的產能-成本曲線
- **風險評估**：需求不確定性對投資回報率的影響

## 四、存貨管理（Inventory Management）

### 4.1 存貨的功能與成本

存貨（Inventory）在供應鏈中扮演緩衝角色，但同時產生成本。

**存貨功能：**
1. **週期庫存**（Cycle Stock）：滿足常規需求
2. **安全庫存**（Safety Stock）：應對需求波動與供應不確定性
3. **季節庫存**（Seasonal Stock）：預見季節性需求變化
4. **交運庫存**（Pipeline Inventory）：在供應鏈各環節的在途庫存

**存貨成本結構：**
- 購買成本（Product Cost）
- 持有成本（Holding Cost）：資金佔用、倉儲、過期損耗，通常為商品成本的20-40%
- 訂購成本（Ordering Cost）：下單、運輸、驗收
- 缺貨成本（Shortage Cost）：失去銷售、客戶流失、補救成本

### 4.2 經濟訂購量模型（Economic Order Quantity, EOQ）

EOQ是經典的庫存優化模型，用於確定最優訂購量。

**基本EOQ公式：**
```
EOQ = √(2DS/H)

其中：
D = 年度需求量
S = 每次訂購成本
H = 年度單位持有成本
```

**範例計算：**
假設某零件年需求量為12,000件，每次訂購成本$50，年度持有成本為$2/件

EOQ = √(2 × 12,000 × 50 / 2) = √600,000 ≈ 775件

**重要指標：**
- 訂購次數（Order per year）= D / EOQ
- 平均庫存（Average Inventory）= EOQ / 2
- 總相關成本（Total Relevant Cost）= (D/EOQ) × S + (EOQ/2) × H

### 4.3 進階庫存模型

**考慮數量折扣的EOQ：** 當供應商提供大宗購買折扣時，需要評估折扣帶來的成本節省是否超過額外的持有成本。

**周期性需求與時變需求：** 使用動態規劃或啟發式方法（如最低成本法）來處理非均勻需求模式。

**安全庫存的確定：**
```
安全庫存 = Z × σ × √LT

其中：
Z = 期望服務水準的標準分數（如99%對應Z≈2.33）
σ = 需求的標準差
LT = 前置時間（天數）
```

### 4.4 準時制（Just-In-Time, JIT）

JIT是一種極端的庫存最小化策略，由Toyota開創。

**核心原則：**
1. **零缺陷**：品質問題必須在源頭消除，不能依賴檢驗
2. **小批量生產**：減少一次性缺陷的影響
3. **頻繁交付**：供應商與生產線之間的緊密協調
4. **前置時間最小化**：精簡流程、標準化、自動化

**實施條件：**
- 穩定且可預測的需求
- 高度可靠的供應商
- 員工的多功能性與高技能
- 信息系統的完整集成

**成功案例：** Toyota能夠將生產線庫存保持在2-3小時產量，相較業界的2-3週產量優勢明顯。

## 五、精實生產（Lean Manufacturing）

### 5.1 精實生產的起源與演進

精實生產源於Toyota生產系統（Toyota Production System, TPS），在2000年代因Womack等人的著作在全球製造業推廣。精實生產不僅是庫存管理方法，而是一套完整的經營哲學。

### 5.2 七大浪費（Seven Wastes）

精實生產識別並消除七種浪費：

| 浪費類型 | 定義 | 範例 |
|---------|------|------|
| **過度生產** | 生產超過立即需求 | 批次過大、預測不準 |
| **等待** | 工作人員或物料的閒置 | 機器故障、排隊等候 |
| **非必要搬運** | 物料無必要的移動 | 佈局不合理、倉儲地點遠 |
| **加工浪費** | 不必要的製程步驟 | 過度檢驗、重複設置 |
| **庫存** | 過量的在製品與成品 | 前置時間長、需求波動 |
| **動作浪費** | 不必要的人工動作 | 人體工程學設計不佳 |
| **缺陷與返工** | 品質問題導致返工 | 製程控制不佳 |

### 5.3 Toyota生產系統的兩大支柱

**第一支柱：自動化（Jidoka）**
- 賦予機器與員工「停止異常」的權力
- 實施品質控制在製程內而非事後檢驗
- 員工可在發現問題時立即停止生產線

**第二支柱：JIT**
- 如前所述的準時制原則
- 與需求同步的拉式系統（Pull System）

### 5.4 拉式系統與看板（Kanban）

看板是實施JIT的核心控制機制。看板卡片（通常為實物或電子標籤）流過生產系統，指示何時生產、運輸或供應物料。

**看板運作邏輯：**
```
生產看板：通知上游工序何時生產
提取看板：指定物料供應商何時運輸
```

**計算看板卡片數量：**
```
看板卡片數 = [日需求量 × (製造提前期 + 提取提前期)] / 容器容量 + 安全係數

典型安全係數：10-15%
```

### 5.5 精實實施方法：價值流圖改善

改善過程通常分為五個步驟：

1. **識別價值**：從客戶視角定義何為增值
2. **價值流映射**：繪製現狀與未來狀態圖
3. **建立流動**：消除批次生產，實現單件流（Single Piece Flow）
4. **實施拉式系統**：從推式改為拉式生產
5. **持續改善**：建立Kaizen（改善）文化

## 六、六標準差（Six Sigma）

### 6.1 六標準差的統計基礎

六標準差是一套系統的質量改善方法，目標是將製程變異控制在極低水準。「六標準差」指製程平均值距規格上下限各6個標準差，不良率為3.4百萬分之一（DPMO）。

**對比三標準差：**
- 三標準差：不良率0.27%（每萬件有27件不良）
- 六標準差：不良率0.00034%（每百萬件有3.4件不良）

### 6.2 DMAIC改善循環

六標準差的核心方法是DMAIC，由五個階段組成：

#### **D - 定義（Define）**
- 識別改善項目與客戶關鍵需求
- 定義現狀績效指標（Baseline Metrics）
- 估計改善的財務影響

#### **M - 測量（Measure）**
- 選擇關鍵性能指標（KPI）
- 進行製程能力分析（Cpk分析）
- 評估測量系統的有效性（MSA - Gage R&R）

**製程能力指數（Cpk）計算：**
```
Cpk = min[(USL - μ)/(3σ), (μ - LSL)/(3σ)]

其中：
USL/LSL = 上/下規格限
μ = 製程平均值
σ = 製程標準差

Cpk ≥ 1.67 對應六標準差水準
```

#### **A - 分析（Analyze）**
- 使用統計工具識別根本原因
- 常用工具：魚骨圖、假設檢定、相關分析
- 確定關鍵X因素（改善的自變數）

#### **I - 改善（Improve）**
- 設計改善方案與試驗
- 使用設計實驗法（Design of Experiments, DOE）優化參數
- 試點測試與驗證

**DOE範例：** 假設要改善注塑成型的尺寸精度，可測試溫度（300°C、320°C、340°C）與壓力（80Bar、100Bar、120Bar）的9種組合，找到最優參數組合。

#### **C - 控制（Control）**
- 建立標準化流程與控制圖
- 培訓員工並持續監控
- 防止改善成果的反彈

### 6.3 六標準差的認證與角色

- **黑帶（Black Belt）**：領導主要改善項目，年可貢獻百萬級美元收益
- **綠帶（Green Belt）**：參與黑帶項目或領導小規模改善
- **黃帶（Yellow Belt）**：具備基本六標準差知識的員工

### 6.4 精實六標準差（Lean Six Sigma）

結合精實生產的速度與六標準差的精確性，形成更強大的改善方法。精實側重流程速度與去除浪費，六標準差側重品質與變異控制。

## 七、全面品質管理（Total Quality Management, TQM）

### 7.1 TQM的演進與定義

TQM是一種涵蓋整個組織的經營哲學，強調品質不是品質部門的職責，而是每個員工的責任。

**關鍵特徵：**
- 客戶導向（Customer Focus）
- 過程導向（Process Orientation）
- 員工賦權（Employee Empowerment）
- 持續改善（Continuous Improvement）
- 數據驅動決策（Fact-Based Decision Making）

### 7.2 品質管理工具與方法

**統計過程控制（Statistical Process Control, SPC）**：
- 使用控制圖（Control Charts）監控製程
- X-R圖用於連續資料，P圖用於不良率
- 當樣本點超出控制限或出現異常趨勢時，視為製程失控

**根本原因分析（Root Cause Analysis）**：
- 五個為什麼法（5 Whys）：逐層問「為什麼」以找到根本原因
- 魚骨圖（Fishbone Diagram）：系統地列舉所有可能原因
- 故障樹分析（Fault Tree Analysis）：從失敗結果反向追溯

### 7.3 ISO品質管理標準

**ISO 9001**：品質管理體系國際標準，涵蓋：
- 領導層承諾與客戶焦點
- 方針與目標的制定與溝通
- 資源與能力管理
- 運作規劃與控制
- 績效評估與改善

## 八、供應鏈營運與銷售與營運規劃（S&OP）

### 8.1 供應鏈營運的目標

供應鏈（Supply Chain）從原材料採購到最終客戶交付的全過程。有效的供應鏈營運追求：

1. **成本最小化**：優化採購、運輸、存貨
2. **服務水準最大化**：高可用性、短交期、準確性
3. **敏捷性與靈活性**：快速應對市場變化
4. **可持續性**：環保與道德責任

### 8.2 採購決策

**自製vs外包決策分析：**
```
自製成本 = 固定成本 + (變動成本 × 年需求量)
外包成本 = 單位採購成本 × 年需求量

選擇成本更低的方案，並考慮質量、風險、戰略重要性
```

**供應商選擇與管理：**
- 多源採購：分散風險但增加複雜性
- 單一源採購：成本優勢但缺乏冗餘
- 分類採購：根據物料的策略重要性與供應風險進行四象限分類

### 8.3 銷售與營運規劃（Sales and Operations Planning, S&OP）

S&OP是一套整合銷售預測、生產計畫與庫存管理的高層管理流程。

**S&OP流程的五個步驟：**

1. **需求規劃**（Demand Planning）
   - 收集銷售、市場與歷史資料
   - 生成初步需求預測

2. **供應規劃**（Supply Planning）
   - 評估現有產能與庫存
   - 確定生產、採購與庫存計畫

3. **中間認可**（Pre-S&OP Meeting）
   - 財務部門審視成本影響
   - 初步協調各職能部門

4. **高層決策**（Executive S&OP Meeting）
   - 副總裁級別決策者參與
   - 平衡成長、利潤與現金流
   - 做出資源配置決策

5. **執行與監控**（Execute and Monitor）
   - 將計畫轉化為具體指令
   - 追蹤績效並定期審視

**S&OP的財務影響：**
根據APICS統計，實施有效S&OP的企業通常能夠：
- 存貨減少10-15%
- 預測準確度提升10-15%
- 交期準確度達95%以上
- 利潤增加2-3%

## 九、服務營運管理

### 9.1 服務營運的特性

服務營運不同於製造，具有獨特特徵：

| 特徵 | 製造業 | 服務業 |
|------|--------|--------|
| **有形性** | 有形產品 | 多數無形 |
| **儲存性** | 可庫存 | 不可庫存（容量浪費） |
| **客戶參與** | 低 | 高 |
| **品質衡量** | 客觀標準 | 主觀感受 |
| **流程可變性** | 低 | 高 |

### 9.2 服務設計與傳遞

**服務藍圖（Service Blueprint）**：
類似於製造的流程圖，但更強調客戶接觸點（Touchpoint）。藍圖包括：
- 客戶行動線
- 前台員工行動線
- 後台支援流程
- 支援系統與資源

**範例：餐廳服務**
```
客戶行動：進入→點餐→等待→用餐→結帳→離開
前台行動：迎接→記錄→確認廚房→上菜→核對→收款
後台流程：採購→存貨→廚房製備→品檢→清潔
```

### 9.3 服務水準與產能管理

**座位預約率**（Booking Percentage）：容量預先銷售的比例
- 航空業：常見95-98%
- 飯店業：常見85-90%

**排隊理論（Queuing Theory）**：
```
L = λ / (μ - λ)  [M/M/1隊列公式]

其中：
λ = 到達率（客人/時間）
μ = 服務率（客人/時間）
L = 平均隊列長度
```

當λ接近μ時，隊列快速增長，體驗惡化。服務營運必須維持充足產能。

### 9.4 服務品質的衡量：SERVQUAL

SERVQUAL模型透過五個維度評估服務品質：

1. **有形性（Tangibles）**：設施、設備、員工外表
2. **可靠性（Reliability）**：準時交付、承諾兌現
3. **反應性（Responsiveness）**：員工的幫助意願與速度
4. **保證性（Assurance）**：員工知識與禮儀
5. **同理心（Empathy）**：個人化關注與理解

## 十、排程與瓶頸管理（Scheduling & Theory of Constraints）

### 10.1 生產排程的目標與限制

生產排程（Production Scheduling）涉及分配工作、機器與員工以完成製造計畫。目標通常包括：

- **最小化製期時間（Makespan）**：完成所有工作的總時間
- **最小化逾期交貨**：確保按期交付
- **最小化設置時間**：減少機器調換品種的停機
- **機器利用率最大化**：提高產能使用效率

### 10.2 排程規則與演算法

**優先規則（Dispatching Rules）**：

| 規則 | 邏輯 | 適用情況 |
|------|------|---------|
| **FCFS** | 先進先出 | 公平性優先 |
| **SPT** | 最短加工時間優先 | 最小化完成時間 |
| **EDD** | 最早交期優先 | 最小化逾期 |
| **CR** | 關鍵比率（(交期-今日)/(剩餘工作時間)) | 平衡效率與準時 |

**案例：五項工作在單一機器上的排程**

| 工作 | 加工時間 | 交期 |
|------|---------|------|
| A | 3天 | 第8天 |
| B | 5天 | 第15天 |
| C | 2天 | 第5天 |
| D | 4天 | 第12天 |
| E | 1天 | 第6天 |

使用SPT規則的順序：E(1) → C(2) → A(3) → D(4) → B(5)
- 製期時間：15天
- 平均完成時間：(1+3+6+10+15)/5 = 7天

### 10.3 Goldratt的制約理論（Theory of Constraints, TOC）

TOC是由Eliyahu Goldratt提出的管理方法論，強調系統中的瓶頸決定了整體產出。

**核心概念：**

1. **瓶頸的識別**：
   - 瓶頸（Constraint）：產能小於需求的資源
   - 非瓶頸（Non-Constraint）：有閒置產能的資源

2. **五聚焦步驟（Five Focusing Steps）**：

   **第一步：識別瓶頸**
   - 監控每個工作中心的利用率
   - 瓶頸的特點：總是繁忙、有待機工作積累

   **第二步：決定如何利用瓶頸**
   - 排除瓶頸內部的浪費與低效
   - 最大化瓶頸的有效產出時間
   - 優先生產高邊際貢獻的產品

   **第三步：使其他資源服從瓶頸**
   - 非瓶頸的產能應受瓶頸約束
   - 上游節點只生產足以供應瓶頸的數量
   - 避免瓶頸前產生堆積

   **第四步：解除瓶頸的限制**
   - 增加瓶頸的產能（購置設備、增加班次、外包）
   - 改進瓶頸的工藝流程

   **第五步：回到第一步**
   - 解除原瓶頸後，新的瓶頸將浮現
   - 持續迭代改善

### 10.4 鼓-緩衝-繩（Drum-Buffer-Rope, DBR）排程系統

DBR是TOC在實務中的實施工具：

- **鼓（Drum）**：瓶頸工作中心，決定整個系統的節奏
- **緩衝（Buffer）**：在瓶頸前設置時間或庫存緩衝，防止瓶頸餓食
- **繩（Rope）**：一套同步機制，確保上游工作進度與瓶頸同步

**DBR的優勢：**
- 大幅減少在製品庫存
- 提高交期準確度
- 最小化瓶頸的等待時間

## 十一、營運管理在現代企業中的應用

### 11.1 智能製造與工業4.0

數位化技術正在革新營運管理：

**物聯網（IoT）與感知器**：實時監控設備狀態、產線績效、庫存位置
**大數據分析**：預測維護、需求預測、異常偵測
**雲端計算**：實現全球供應鏈的可視化與協調
**人工智慧**：自動化決策、機器學習優化排程

### 11.2 可持續與綠色營運

- **減少廢棄物**：精實生產與循環經濟原則
- **能源效率**：優化生產工藝、使用可再生能源
- **責任採購**：評估供應商的環保與勞工標準

### 11.3 敏捷供應鏈

在疫情與地緣政治風險下，企業需要：
- 多元化供應商與製造地點
- 快速響應市場變化的靈活流程
- 與供應商的深度合作與信息共享
