# 談判與商業溝通：練習與自測

---

## 一、是非題（True/False）

請判斷以下敘述是否正確。

**1. 分配型談判的目標是讓一方獲得所有利益，另一方獲得最少。**

答：**假（False）**

解說：分配型談判是指資源固定，但雙方都試圖獲得最好的分配。目標是在既定餅內尋求最優分配，而非讓一方贏得所有。整合型談判才是通過創新來擴大整體價值。

---

**2. BATNA是指你在談判中能要求的最高價。**

答：**假（False）**

解說：BATNA（最佳替代方案）是指談判失敗時的替代方案。它決定了你的保留價格（你能接受的最低/最高價），但不是你能要求的最高價。最高要求應該高於BATNA，以留下談判空間。

---

**3. 在談判中隱瞞資訊通常比坦白更能獲得優勢。**

答：**假（False）**

解說：在整合型談判中，隱瞞資訊會阻礙創意解決方案的發現。在分配型談判中雖然隱瞞可能短期有利，但會損害長期信任。研究表明，資訊透明反而能幫助雙方找到雙贏方案。

---

**4. 哈佛談判法中，「立場」和「利益」是同一個概念。**

答：**假（False）**

解說：立場是表面要求（「我要$50,000」），利益是深層原因（「我需要$50,000來支付孩子教育」）。理解利益而非立場，是原則性談判的核心。

---

**5. 在高PDI（權力距離）文化中，直接與基層員工談判比與高管談判更容易達成協議。**

答：**假（False）**

解說：在高PDI文化中，等級制度受重視，基層員工往往沒有決策權。應該通過高層或領導者進行談判，以確保協議的有效性與執行力。

---

**6. 原則性談判中，使用客觀標準比使用談判籌碼更有效。**

答：**真（True）**

解說：客觀標準（如市場價格、行業標準、先例）基於事實而非權力，對雙方都有說服力。依賴談判籌碼容易導致對立，而客觀標準能幫助雙方達成共識。

---

**7. Thomas-Kilmann模式中，「競爭」風格在所有情況下都應該避免。**

答：**假（False）**

解說：競爭風格在應對緊急情況、原則性分歧或自衛時是必要的。不同情況需要不同的衝突管理風格。

---

**8. 非語言溝通佔據總溝通內容的55-93%，所以在談判中應該忽視口頭內容，只關注肢體語言。**

答：**假（False）**

解說：非語言溝通確實很重要，但這不意味著應該忽視口頭內容。非語言與口頭溝通應該協調一致。當兩者不一致時，聽者會相信非語言信號更多，但都應該關注。

---

**9. 故事敘述在商業溝通中的效果不如統計數據有效。**

答：**假（False）**

解說：故事比統計數據更容易被記住和相信。研究表明，故事+數據的組合最有效。故事提供情感連接，數據提供證據。

---

**10. 視訊會議中，看著屏幕上的自己比看著鏡頭讓人更舒適。**

答：**假（False）**

解說：看屏幕上的自己看起來像在低頭，給人不專注或沒有信心的印象。應該看著鏡頭，即使感到不自然。

## 二、選擇題（Multiple Choice）

**1. 根據Fisher與Ury的談判理論，以下哪項不是原則性談判的四大原則？**

A) 分開人與問題
B) 著眼於利益而非立場
C) 優先滿足強勢方的需求
D) 產生互利選項

**答：C**

解說：原則性談判的四大原則是：(1)分開人與問題，(2)著眼於利益，(3)產生互利選項，(4)堅持客觀標準。優先滿足強勢方反而是對抗性談判的特點，不符合原則性談判。

**2. 你想買一台新電腦，心理價位最多$5,000。賣家開價$6,500。這種情況下，哪項描述最準確？**

A) 你的保留價格是$6,500
B) 你的BATNA是保留現在的舊電腦
C) 談判區間是$5,000-$6,500
D) 目前沒有談判區間，因為雙方立場不重疊

**答：B**

解說：你的保留價格（能接受的最高價）是$5,000。如果賣家堅持$6,500，你的BATNA是保留舊電腦。目前沒有重疊，但如果賣家願意降價到$5,000-$5,500，就會形成談判區間。

**3. 在跨文化談判中，以下哪項不是高不確定性規避文化（高UAI）的特點？**

B) 對不確定性感到不安
C) 願意接受創新與模糊的條款
D) 重視法律保障與規則

**答：C**

解說：高UAI文化傾向於詳細協議、規則與確定性。低UAI文化反而願意接受模糊、創新與靈活性。

**4. Aristotle修辭三角中，「Ethos」主要指什麼？**

B) 說話者的品德與信譽
C) 說話者的邏輯論證

解說：Ethos涉及演講者的可信度、專業性與真誠度。Pathos涉及情感，Logos涉及邏輯。

## 四、情境分析（Scenario Analysis）

### 情境1：薪酬談判

**背景：** 你在公司工作了2年，成績突出。你知道市場上同等職位的薪資是年60萬，但你現在只賺50萬。你決定向老闆要求加薪。

**場景：** 你的老闆說：「公司今年業績不太理想。我無法給你加薪。」

**問題：**

1. 在這個談判中，你的BATNA是什麼？
2. 你的保留價格應該是多少？
3. 如果老闆堅持不加薪，但願意提供其他福利（如額外休假、遠程工作機會、股票期權），這如何改變談判策略？

**建議答案：**

1. **BATNA：** 你的BATNA包括：
   - 接受現在的50萬
   - 尋找新工作（市場薪資60萬）
   - 轉職到其他部門（薪資可能55萬）

   最強的BATNA是找新工作賺60萬。這給了你談判的底氣。

2. **保留價格：** 基於BATNA，你的保留價格應該在58-60萬之間（略低於市場價，以保留留在原公司的靈活性）。你可以要求的起價是62-65萬（高於市場，留下談判空間）。

3. **多元福利的談判：** 當薪資加幅受限時，可轉向其他福利：
   - **額外休假：** 可節省你的生活成本或改善生活品質，相當於隱性加薪
   - **遠程工作：** 可降低交通成本與時間，相當於隱性加薪
   - **股票期權：** 如果公司前景好，可能是長期價值，但風險也高
   - **簽約獎勵：** 立即現金，可補償薪資差額

   你可以說：「如果薪資短期難以調整，我們能否談：(1)下次半年評審時重新審視，(2)遠程工作每週2天，(3)額外5天年假？」

   這樣，即使薪資沒增加，你也獲得了實質福利，談判不失敗。

### 情境2：跨部門專案資源爭奪

**背景：** 你是市場部經理，計劃一個大型品牌活動。你需要IT部門的技術支援（3個月，4名工程師）。但IT部長告訴你，他們已經承諾給另一個優先級更高的專案。

**場景：** IT部長說：「抱歉，我們無法同時支援兩個大專案。你的專案要延後3個月。」

**問題：**

1. 分析這個談判中的利益衝突。
2. 如何從固定餅（「誰先誰後」）轉向整合談判（「如何都滿足」）？
3. 設計一個可能的互利方案。

**建議答案：**

1. **利益分析：**

   市場部的利益：
   - 品牌活動按時進行，趕上市場季節（如Q4假期季）
   - 品牌曝光與銷售轉化
   - 部門績效目標

   IT部的利益：
   - 優先級高的專案不被打擾（系統穩定性、公司核心業務）
   - 資源可控（避免多專案衝突）
   - 避免加班與員工疲勞

2. **從固定餅到整合餅的轉變：**

   固定餅思維：「到底誰先進行？」→ 必然有人延後

   整合思維：「有沒有辦法讓兩個專案都進行，只是時間或資源配置不同？」

   可能的創意問題：
   - 市場專案能否分階段進行，分散IT工作量？
   - IT專案能否外包部分工作，釋放內部資源？
   - 能否共享部分技術資源（如一個架構師 part-time）？
   - 市場專案能否使用現成工具（低代碼/無代碼），減少定制化IT工作？

3. **互利方案示例：**

   你可以提議：

   ```
   方案：並行但分階段推進

   A. 市場專案的第一階段（第1-6週）只需要2名IT工程師
      做基礎架構建設，與IT專案不衝突。

   B. IT專案完成（預計第8週）後，4名IT工程師全力
      支援市場專案的第二階段（第9-12週）。

   C. 為補償市場專案的延遲，我們：
      - 提供IT部門額外的IT設備預算（他們正好想更新伺服器）
      - 承諾在下一個品牌活動中，提前6個月預訂資源

   D. 建立跨部門的專案管理機制，定期同步進度，
      避免資源衝突。
   ```

   這樣，IT部不用犧牲優先專案，市場部也在可接受的時間內完成專案，並且雙方建立了更好的協作模式。

## 五、實作練習（Role-Play Scenarios）

### 練習1：供應商降價談判

**角色設定：**
- 你是製造企業的採購經理，向現有供應商要求降價15%
- 你的供應商是一個中等規模的零部件製造商，與你合作3年

**背景：**
- 目前價格：每單位$100
- 市場競爭對手價格：$85/單位
- 你的採購量：每年100萬單位
- 供應商承諾：3年內交付1,500萬單位

**供應商的考慮：**
- 降價15%意味著利潤從20%降到5%
- 供應商的年收益會從2,000萬美元降到1,000萬美元
- 但丟失這個大客戶也會嚴重打擊他們的財務狀況

**你的任務：**

1. 在**固定餅談判**中，如何從市場價格角度進行強硬但合理的談判？
2. 在**整合談判**中，提出一個可能的互利方案，不只依賴降價。

**答案示例：**

**固定餅談判（市場導向）：**

```
採購經理：「我之前很感謝你們的合作。但現在市場
已經改變。我看到3個競爭對手可以提供$85的價格。
為了繼續與你們合作，我需要把價格調到$85-$90。」

供應商：「$85太低了，我們無法接受。$95是我們的最低。」

採購經理：「$95仍高於市場。但我理解你們的成本。
我提議$90，但我需要你們給我一個月的付款期限
（現在是現金結算）。這樣能幫助我的現金流嗎？」

供應商：「$90很勉強，但如果有30天付款期，
我們的財務成本降低。我們可以接受$92。」

採購經理：「成交。但我還有一個條件：如果你們
能在某些規格上優化，幫我降低我的製造成本，
我願意在三年合作中維持$92的價格。」
```

**整合談判（創意方案）：**

```
採購經理：「我知道降價對你們很困難。讓我們共同
想想，除了降價外，還有什麼辦法能提升我們雙方
的價值。」

供應商：「我們很願意談。但你們的需求已經很明確了。」

採購經理：「不一定。我們的真實利益是：
(1)降低成本（但不是唯一方式）
(2)確保品質與供應穩定性
(3)長期合作夠經濟高效

你們的利益是什麼？」

供應商：「我們想：
(1)維持利潤以投資設備與技術
(2)確保訂單穩定（不怕被拋棄）
(3)有空間創新」

採購經理：「很好。那我們能否嘗試：

方案A：年度訂購量保證
- 我承諾每年最少訂購80萬單位（現在是變動的）
- 作為回報，你們降價到$95

方案B：共同成本優化
- 我們一起分析你們的成本結構
- 幫你們找出降本機會（如材料採購、工藝改進）
- 降本收益對半分享。如果降到$90，我們各得利益

方案C：長期合約
- 簽署5年合約，每年根據通脹調整（而非市場波動）
- 給你們長期收益穩定性
- 同時給我們長期價格預測

你們覺得哪個方向可行？」
```

在整合談判中，即使最終價格不是$85，但通過確保訂單穩定、成本優化空間、長期合作，雙方都獲得了實質性好處。

### 練習2：遠程團隊衝突與溝通

**場景：**

你是遠程團隊的經理。你的團隊分佈在5個時區。最近，日本員工Tanaka抱怨，美國員工Josh的回覆郵件總是簡短冷淡，讓他覺得被忽視。Josh則感到困惑，他認為自己已經及時回覆了所有郵件。

文化背景：
- 日本（高PDI、高UAI、高集體主義）：重視詳細溝通、關係建立、尊重與細心
- 美國（低PDI、低UAI、高個人主義）：強調效率、快速溝通、直來直往

**問題：**

1. 這個衝突的根本原因是什麼？
2. 使用Thomas-Kilmann的哪種衝突管理方式最適合？

**答案示例：**

1. **根本原因分析：**

這是典型的跨文化溝通誤解。

Tanaka的期望：
- 詳細的郵件（展示關心）
- 問候與社寒暄（建立關係）
- 邀請意見與討論（體現尊重）

Josh的風格：
- 簡潔的郵件（表示效率）
- 直接回答問題（不浪費時間）
- 假設沒有問題就是OK（直接風格）

從Tanaka的視角：Josh的簡短意味著他不在乎合作。
從Josh的視角：Tanaka的詳細要求變得冗長浪費時間。

實際上，雙方都沒有問題，只是文化期望不同。

2. **應該使用「合作 (Collaborating)」風格：**

- **主張性高：** 明確指出這是跨文化誤解，需要解決
- **合作性高：** 幫助雙方理解彼此的背景與意圖
- 不應該用「競爭」(傷害關係)、「退讓」(沒有真正解決)、或「迴避」(問題堆積)

3. **具體處理步驟：**

**第一步：一對一訪談（分別與Tanaka和Josh進行視訊通話，不是郵件）**

與Tanaka：
```
「Tanaka，我聽說你覺得Josh的溝通有些冷淡。
我想理解你的感受。能具體說說是什麼時候發生的嗎？」

Tanaka：「是的。他回覆我的郵件只有一句話『已收到』
或『OK』。我覺得他不想真正與我合作。」

經理：「我理解。在你的文化背景中，詳細的溝通
表示關心與尊重。但Josh來自美國文化，簡潔是
表示效率與信任。這不是他不關心你，而是
溝通風格不同。」
```

與Josh：
```
「Josh，我想和你聊聊遠程團隊的溝通。
Tanaka感到你的郵件很簡短。這是有意的嗎？」

Josh：「沒有。我只是想快速回覆他，
不浪費他的時間。為什麼有問題嗎？」

經理：「Tanaka來自日本，他期待更詳細的郵件
和關於專案的討論。簡短的郵件讓他覺得
被忽視。你願意調整一下溝通方式嗎？」

Josh：「哦，我沒意識到這一點。我可以試試。」
```

**第二步：團隊會議（視訊，確保所有人參與）**

```
經理：「我想與大家分享一個重要話題：
跨時區、跨文化的遠程團隊溝通。

大家來自不同文化：日本、美國、印度、英國。
我們的溝通風格會很不同。有些人喜歡詳細，
有些人偏好簡潔。有些人在郵件前想深思，
有些人快速反應。

這些差異都是寶貴的。但如果我們不理解
彼此的背景，就會誤解。

所以，我希望我們一起建立一個『遠程溝通契約』：

1. 重要決定不用郵件，用視訊會議
2. 郵件中，添加『背景簡述』幫助全球同事理解
3. 回覆郵件時，確認收到並給出初步意見
   （即使不是最終決定）
4. 定期一對一視訊，不只談工作

大家覺得可行嗎？」
```
