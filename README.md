## index.html 運作方式

`index.html` 是一個不依賴任何建置工具的單頁應用程式（SPA），唯一的外部依賴是 [marked.js](https://github.com/markedjs/marked)（用於將 Markdown 轉換為 HTML）。

**核心流程：**

1. **資料來源**：啟動時透過 `fetch` 載入 `README.md`，解析其中 `### 主題名稱` 和 `- [連結](路徑) — 描述` 的結構，分別取得各主題與跨主題整合分析的資料
2. **首頁呈現**：首頁有四個頁籤 — 學習路線圖、各主題分析（卡片式列表）、跨主題整合分析、術語總表。學習路線圖和術語總表分別從 `src/學習路線圖.md` 和 `src/術語總表.md` 載入後以 marked.js 渲染
3. **主題閱讀器**：點擊任一主題卡片後進入閱讀器視圖，透過頁籤切換該主題下的六份文件（專業版、白話版、案例分析、練習與自測、進一步學習、互動圖表）。Markdown 文件以 `fetch` 載入並用 marked.js 即時渲染；互動圖表（`.html`）則以 `<iframe>` 嵌入
4. **全文搜尋**：背景非同步載入所有 Markdown 文件內容建立索引，支援即時搜尋主題名稱、術語、以及文件全文
5. **路由**：使用 `hash` 路由（如 `#供應鏈管理`），支援瀏覽器上一頁/下一頁導覽

**新增或修改主題時**，只需：
- 在 `src/` 對應資料夾內新增/編輯 Markdown 或 HTML 文件
- 在 `README.md` 中新增/更新該主題的連結列表（路徑需包含 `src/` 前綴）
- `index.html` 會自動解析並呈現，無需額外設定
