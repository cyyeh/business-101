# 營運管理：白話版

想像一下，你開了一家夜市紅豆餅攤。每天要決定進多少紅豆、多少麵粉；要安排工人的班次；要想辦法讓顧客快速拿到熱騰騰的餅。這些日常的決定，其實就是**營運管理（Operations Management）**。

---

## 一、什麼是營運管理？就是把東西做好

營運管理就是把資源（人、機器、材料、錢）轉變成產品或服務的藝術。

換句話說：
- **製造業**：把原料變成產品（例如：蘋果公司把零件組裝成iPhone）
- **服務業**：提供顧客體驗（例如：餐廳把食材煮成美食，航空公司載乘客到目的地）

好的營運管理能帶來三個好處：
1. **成本低** - 不浪費錢、不浪費時間
2. **品質好** - 產品穩定可靠
3. **速度快** - 快速交付給客戶

---

## 二、流程設計：怎樣把事情做好

### 流程就像食譜

一個流程就像烹飪食譜。你有原材料（投入），經過一系列步驟（製程），最後得到完成品（產出）。

例如麻婆豆腐的流程：
```
豆腐 → 切豆腐 → 炒肉末 → 加辛香料 → 加豆腐 → 勾芡 → 盛盤 → 上桌
```

### 看清楚每一步：流程映射

想改善流程，首先要把每一步都看清楚。這叫做**流程映射（Process Mapping）**。

一個例子：餐廳的顧客服務流程

```
顧客進門(2分鐘) → 等位(2-30分鐘) → 入座(2分鐘)
→ 看菜單(5分鐘) → 點餐(2分鐘) → 廚房製作(15分鐘)
→ 上菜(1分鐘) → 用餐(30分鐘) → 結帳(3分鐘)
→ 離開(1分鐘)

總耗時：約60分鐘
```

餐廳老闆看到這個流程後想：「顧客在等位和廚房製作上花最多時間。我是不是應該：
1. 加快上菜速度（請更多廚師）？
2. 增加座位數，減少等待時間？
3. 提供線上預訂，讓顧客不用排隊？」

這就是改善！

### 找出浪費的地方

在廚房裡，你可能會發現：
- 廚師經常找不到醬瓶，浪費時間翻箱倒櫃（浪費）
- 每天都有一堆剩菜沒賣出去（浪費）
- 上午生意差，員工閒著；下午忙得亂七八糟（浪費）

**流程重新設計**就是根本性地改造這些流程，消除浪費。例如有些科技公司透過改變信用審批流程，把原本要7天的流程縮短到1小時。

## 三、產能規劃：做多少才好

### 什麼是產能

**產能（Capacity）**就是你最多能做多少東西。

例如：
- 你的夜市攤位，一小時最多能做100個紅豆餅 = 100個/小時的產能
- 餐廳有50個座位，每位顧客平均用餐2小時 = 一整天可服務 25 × 開店時數 位顧客
- 理髮店一位設計師一小時剪一個頭 = 1個頭/小時的產能

### 產能的三個層級

1. **理論產能**：在完美條件下能做多少
   - 例如：機器24小時不停機運轉

2. **有效產能**：實際上能做多少（考慮休息、保養）
   - 例如：機器只運轉16小時（8小時保養與換班），產能變為理論產能的2/3

3. **實際產能**：真的做了多少（考慮所有現實狀況）
   - 例如：機器故障了、員工請假了、品質問題要返工

## 四、存貨管理：囤多還是囤少

### 存貨的困擾

想像你是便利商店老闆。便當是熱銷品，但是：

**囤太多的問題：**
- 如果你一次進500個便當，但只賣出100個，剩下的過期了，虧大了！
- 便當冷掉也不好吃，客人不買

**囤太少的問題：**
- 顧客想買卻沒有，客人生氣，以後不來了
- 失去銷售機會，損失利潤

**所以要找平衡點。**

### 經濟訂購量：簡單說就是多久進一次貨

**經濟訂購量（EOQ - Economic Order Quantity）**是一個計算最優進貨量的方法。

簡單邏輯：
```
一次進100個便當 → 進貨次數多 → 運費多 → 成本高
一次進1000個便當 → 進貨次數少 → 運費少 → 但庫存多，容易壞 → 成本也高

最優的點在中間某個數字
```

實際計算（簡化版）：
如果你一年賣12000個便當，每次進貨要付500元運費，囤貨一個便當一年要花2元的冷藏費：

```
最優進貨量 = √(2 × 12000 × 500 / 2) ≈ 775個

意思是：一次進775個便當最便宜，
一年進貨次數 = 12000 ÷ 775 ≈ 15次
```

### 安全庫存：防止缺貨的保險

有時候客人來得特別多，你的便當不夠。為了防止缺貨，你會多囤一點。這叫**安全庫存**。

安全庫存要考慮：
- 需求有多波動（有時賣50個，有時賣200個）
- 供應商要多久才能補貨（3天內補到？1週？）

### JIT（準時制）：富人的奢侈品

想像有個便利商店，便當永遠剛好新鮮，不會過期。怎麼做到的？

答案是與供應商**天天協力**。每天凌晨，供應商根據前一天的銷售送來剛好的數量。不多不少。

這叫**JIT（Just-In-Time，準時制）**。

**優點：**
- 沒有過期損失
- 永遠新鮮

**缺點：**
- 需要可靠的供應商（他要天天準時送）
- 如果供應商遲到或供應不足，你就缺貨了
- 系統必須非常協調，任何地方出錯都會影響

**誰用JIT？** 像Toyota這樣的大企業。他們與供應商深度合作，甚至供應商就設在工廠附近，確保每2小時就能補貨。

## 五、精實生產：消除所有的浪費

### 七大浪費，你家廚房就有

**精實生產（Lean Manufacturing）**最核心的想法就是：**消除浪費**。

什麼是浪費？來看看廚房例子：

| 浪費類型 | 廚房例子 |
|---------|--------|
| **過度生產** | 一次炒了100盤菜，但只有10位客人，剩下90盤冷掉 |
| **等待** | 廚師等著水燒開、等著醬油從老遠地方拿來 |
| **搬運浪費** | 廚房設計不好，要煮菜時還要走來走去找鍋子 |
| **加工浪費** | 一道菜要檢查5次品質，但其實檢查1次就夠 |
| **庫存** | 冰箱堆滿剩菜，有些還要丟掉 |
| **動作浪費** | 廚師要站在危險高度夠不著調味罐 |
| **缺陷** | 菜做錯了，要重新做一次 |

精實生產的目標是一個一個消除這些浪費。

### Toyota怎樣做的

Toyota之所以聞名於世，就是因為他們把精實生產做到極致。

**兩個法寶：**

**1. 自動化（Jidoka）：給員工權力停止生產線**
- 如果員工發現有問題（例如零件錯了），可以按一個按鈕，整條生產線停止
- 立刻修復，然後再開始
- 這樣就不會生產出1000個有缺陷的產品，然後才發現問題

**2. 準時制（Just-In-Time）：如前所述**

### 看板系統：像菜單一樣

**看板（Kanban）**是一個簡單卻有力的工具。它就像一個菜單卡片。

例如：
```
便利商店的補貨看板：
┌─────────────────┐
│ 紅豆奶茶        │
│ 需要補貨！      │
│ 目前庫存: 5罐   │
│ 建議補到: 50罐  │
└─────────────────┘
```

供應商看到這張卡片，就知道要補貨。補好後，卡片回到初始位置。系統簡單透明，不需要複雜的電腦系統。

## 六、六標準差：讓品質穩定到極致

### 什麼是六標準差

你開咖啡廳，每杯咖啡都要250ml。但實際上：
- 這杯245ml
- 那杯255ml
- 另一杯248ml

變來變去，有些顧客會抱怨：「我買的咖啡怎麼比昨天少？」

**六標準差（Six Sigma）**就是一套方法，讓你的咖啡每杯都非常接近250ml。不是「大約」接近，而是在統計上99.99966%的時間都在規格內。

簡單說：如果你一百萬杯咖啡，只有3杯可能出錯。

### DMAIC五步改善

做到六標準差，需要跟著五個步驟：

**D - 定義（Define）**
- 決定要改善什麼（例如：咖啡的分量）
- 設定目標（例如：從目前的90%準確率提升到99.99%）

**M - 測量（Measure）**
- 量測現在的狀況
- 例如：連續測量100杯咖啡的分量
  - 平均：248ml
  - 有時少到240ml，有時多到256ml

**A - 分析（Analyze）**
- 找出根本原因
  - 是磅秤不準？是員工技術不夠？是咖啡機故障？

**I - 改善（Improve）**
- 測試改善方案
  - 例如：試試自動咖啡機 vs 手動倒 vs 用量匙
  - 看哪個最準確

**C - 控制（Control）**
- 確保改善後的狀況維持
- 例如：每天檢查5杯，確保都在規格內
- 如果有偏差，立刻調整

## 七、品質管理：不是檢驗，是預防

### 傳統vs現代的想法

**舊的做法**：
```
生產 → 檢驗 → 發現壞品 → 丟掉或返工 → 虧錢
```

**現代的做法**：
```
設計好流程 → 仔細執行 → 監控過程 → 預防問題 → 不會有壞品
```

### 統計過程控制：畫個圖就知道狀況

想像你每天都量測產品的某個尺寸，畫成一個圖表：

```
尺寸 mm
10.2 |         ⚠️      發現：這個點超出了正常範圍！
10.1 |    • •  •  •     ← 這些點都正常
10.0 |  • • • • • •
 9.9 |  •  •  •  •
     |_______________
     日期：1 2 3 4 5
```

如果你看到某個點超出了「控制界限」，就表示有問題，要立刻調查。

這叫**統計過程控制（SPC）**。好處是：你能及早發現問題，不會讓不良品流出去。

### 根本原因分析：不斷問「為什麼」

假設便利商店的便當銷售下滑。怎樣找出原因？

```
為什麼便當銷售下滑？
→ 因為顧客說便當不新鮮

為什麼不新鮮？
→ 因為便當中午就做，賣到晚上還沒出完

為什麼賣不完？
→ 因為進貨太多

為什麼進貨太多？
→ 因為沒有根據前一天的銷量調整

為什麼沒有調整？
→ 因為沒有追蹤銷售資料
```

**解決方案：** 每天追蹤銷售，根據前一天調整次日進貨量。

這就是**根本原因分析**，也叫**5個為什麼法**。

## 八、供應鏈管理：從原料到餐桌

### 供應鏈就像菜的旅程

想像一個披薩的旅程：

```
農場的小麥
   ↓
麵粉廠
   ↓
披薩店的倉庫
   ↓
披薩店的廚房
   ↓
你的嘴巴
```

這整個過程，從種小麥到你吃到披薩，就是**供應鏈（Supply Chain）**。

### 採購決策：自己做還是買別人做的

披薩店老闆可能遇到這種決定：

**自己做披薩麵團？**
- 自己做的好處：新鮮、品質可控、利潤更高
- 自己做的壞處：要租廚房、買設備、聘員工

**買現成的麵團？**
- 買的好處：省人力、省空間、成本低
- 買的壞處：品質看供應商、沒有差異化

怎麼決定？計算成本：

```
自己做的成本 = 固定成本（房租、設備、工資）+ 變動成本（材料×銷售數量）

外購成本 = 每個麵團的價格 × 銷售數量

比較哪個便宜
```

### 銷售與營運規劃：協調生產和銷售

想像一個大公司：
- **銷售部** 想：「今年可以賣100萬個！」
- **生產部** 說：「我們產能只有80萬個」
- **財務部** 想：「這會虧錢嗎？」

三個部門吵起來。怎麼辦？

**銷售與營運規劃（S&OP）** 就是讓三個部門坐在一起，討論：

1. 真實的銷售預測是多少？
2. 我們現在的產能能支援嗎？
3. 要不要增加產能（買新機器）？
4. 增加產能的利潤回報值不值得？
5. 做個平衡的計畫，讓大家都同意

做好S&OP的公司通常能：
- 減少20%的庫存
- 增加客戶滿意度
- 增加利潤

## 九、服務營運：你無法庫存一杯咖啡

### 製造和服務的根本差異

**製造業**：做好產品後，放在倉庫裡，慢慢賣
```
生產 → 倉庫 → 賣出去
      ↑
    可以等待
```

**服務業**：服務沒辦法庫存，必須即時提供
```
顧客來 → 立刻服務 → 完成
      ↑
    無法等待或提前做
```

例如：
- 咖啡廳關門前5分鐘，你不能說「預先做100杯咖啡放著」，因為咖啡會冷掉
- 航班起飛，座位沒坐滿，那個收益就永遠消失了
- 理髮師有時間卻沒客人，那段時間的收入就虧損了

所以，**服務營運的核心就是：平衡產能和需求。**

### 服務設計：每個接觸點都重要

去餐廳吃飯，你會經過很多「接觸點」：

```
進門 → 被迎接 → 點餐 → 等待 → 上菜 → 用餐 → 結帳 → 回家
 ↑      ↑      ↑      ↑      ↑      ↑     ↑
每一步都影響你的體驗
```

一家好餐廳會精心設計每一個接觸點：
- **進門**：燈光溫暖、音樂舒服、員工親切
- **等位**：提供舒適座位和茶水，讓顧客不覺得等待時間長
- **點餐**：菜單清楚、推薦合理、下單快速
- **上菜**：餐具乾淨、菜色漂亮、溫度剛好
- **結帳**：快速、無誤、微笑

這叫**服務藍圖**：從顧客的視角設計整個體驗。

### 排隊的科學

為什麼在銀行排隊感覺特別久，但在百貨公司排隊感覺沒那麼久，即使實際時間一樣？

- **銀行排隊**：看著牆壁、看著天花板、無聊
- **百貨公司排隊**：可以看商品、聽音樂、有視覺刺激

所以好的服務會：
- 提供視覺刺激（看電視、逛展示品）
- 提供資訊（隊伍還要等5分鐘）
- 讓等待不感到無聊

這是**排隊理論**的應用。

## 十、瓶頸管理：找到限制，針對它改善

### 每個系統都有瓶頸

想像一個披薩店：
- 老闆一人做披薩
- 有一個助手切菜
- 有一個員工外送

```
做披薩（15分鐘/個） ← 瓶頸！（最慢）
   ↓
切菜（2分鐘/個）
   ↓
外送（30分鐘往返）

假設來了10個訂單：
- 老闆需要150分鐘（10個 × 15分鐘）才能全部做完
- 助手在做前10個時就閒著了
- 外送員也在等
```

瓶頸在**老闆做披薩**。老闆快不了，整個店就快不了。

### 解決瓶頸的五個步驟

**第一步：找出瓶頸**
- 找出哪個環節最慢、最忙

**第二步：最大化瓶頸的效率**
- 老闆不要浪費任何時間
- 不要離開製作台
- 準備好所有材料

**第三步：其他部分都要配合瓶頸**
- 助手只做足夠供應老闆的菜
- 不要做太多，堆積浪費

**第四步：突破瓶頸的限制**
- 雇第二個做披薩的
- 或買更快的披薩爐
- 或簡化披薩做法

**第五步：回到第一步**
- 解決了老闆的瓶頸後，新的瓶頸會出現
- 例如：現在外送變成瓶頸（只有一個人外送）
- 再針對新瓶頸改善

### 鼓、緩衝、繩系統

瓶頸管理的工具叫**鼓-緩衝-繩（Drum-Buffer-Rope）**：

- **鼓**：瓶頸，決定整體步調（如老闆做披薩的速度）
- **緩衝**：在瓶頸前存一點庫存，確保瓶頸不會被餓死（如預先切好的菜）
- **繩**：確保上游同步（如助手的進度和老闆同步，不要超前或落後太多）

## 十一、營運管理在你身邊

### 你知道嗎？

- **星巴克**：用標準化菜單和流程，讓全世界的咖啡味道一致
- **麥當勞**：精確到秒的製作時間，確保食物品質與速度
- **亞馬遜**：自動化倉庫和智能排程，縮短配送時間
- **Zara服裝**：快速供應鏈設計，新款服裝2週內就能上架
- **Netflix**：用大數據預測你想看的影片，優化内容製作

### 未來的營運管理

隨著科技進步：
- **機器人**在工廠裡做更多工作
- **AI預測**需求，自動調整生產
- **物聯網感知器**在每個產品、每個環節，即時回報狀況
- **無人機**配送快遞

但無論多先進，核心思想沒變：
**用最少的資源，做出最好的產品或服務，最快速地交給客戶。**
